Prerequisites

  • Node.js v20 or higher
  • RAM: 4GB minimum, 8GB+ recommended for local embeddings

Installation

git clone <your-repo-url>
cd neural-nexus
npm install

Starting Neural Nexus

Run everything at once:

npm run nexus

This starts Qdrant (port 5304), the API server (port 8008), and the dashboard (port 6969).

Configuration (.env)

API Server

  • PORT — Default 8008
  • NEXUS_PASSWORD — Dashboard admin password

Qdrant

  • QDRANT_URL — Default http://localhost:5304
  • QDRANT_COLLECTION — Default neural_nexus_memories

Embeddings

  • EMBEDDING_MODEL — Default Xenova/bge-small-en-v1.5
  • EMBEDDING_DEVICE — cpu or cuda

Authentication

  • AUTH_ENABLED — Set true to enable JWT auth
  • AUTH_SECRET — Secret key for signing JWTs
  • AUTH_TOKEN_EXPIRY — e.g. 30d

Memory Tuning

  • SIMILARITY_THRESHOLD — Default 0.95. How close two memories must be to merge.
  • RECALL_THRESHOLD — Default 0.01. Minimum relevance score to return a memory.
  • DECAY_LAMBDA — Default 1e-10. Lower = memories last longer.
  • HYBRID_ALPHA — Default 0.7. Balance between vector and keyword search.

Authentication

All endpoints except /health and /auth/login require authentication via one of:

  • Authorization: Bearer <token> — JWT from login
  • X-Api-Key: sk-xxxxxxxx — API key for agent integrations

Token Scopes

  • memory:read — recall, export
  • memory:write — store, import
  • memory:update — reinforce
  • memory:delete — delete memories
  • admin:users — manage users
  • admin:tokens — manage tokens

Auth Endpoints

POST/auth/login

// Request
{ "username": "admin", "password": "yourpassword" }
// Response
{ "token": "eyJ..." }

POST/auth/token

// Request
{ "userId": "uuid", "description": "My Agent", "scopes": ["memory:read", "memory:write"] }
// Response
{ "token": "eyJ..." }

GET/auth/tokens

List active tokens. Query param: userId (admin only).

DELETE/auth/tokens/:tokenId

Revoke a token. Requires admin:tokens scope.

Core Endpoints

POST/store

// Request
{ "text": "User prefers dark mode", "category": "preference" }
// Response
{ "status": "stored" }

POST/recall

// Request
{ "query": "What are the user's preferences?", "limit": 5 }
// Response
{ "memories": [{ "id": "...", "text": "...", "score": 0.91, "category": "preference" }] }

POST/reinforce

{ "memory_id": "uuid", "strength_adjustment": 0.05 }

GET/audit

Returns the memory replacement audit log.

GET/health

{ "status": "ok", "components": { "server": {...}, "qdrant": {...} } }

GET/config

Returns live server configuration values.

Admin Endpoints

GET/admin/users

POST/admin/users

{ "username": "alice", "password": "secret" }

DELETE/admin/users/:id

GET/admin/api-keys

POST/admin/api-keys

{ "name": "my-agent", "model_name": "claude-sonnet-4-6" }

DELETE/admin/api-keys/:id

GET/admin/export

Export all memories as NDJSON.

POST/admin/import

Import memories from NDJSON. Supports streaming for large files.

Internal Metrics

Operational stats are available through the main application endpoints:

  • GET /health — live service and performance snapshot
  • GET /admin/stats/categories — category breakdown
  • GET /admin/stats/merges — merge history summary

MCP Server (Claude / Cursor / VS Code)

Add to your claude_desktop_config.json or IDE MCP settings:

{
  "mcpServers": {
    "neural-nexus": {
      "command": "node",
      "args": ["/absolute/path/to/dist/src/mcp.js"]
    }
  }
}

The agent will have access to recall_memory and store_memory tools automatically.

Direct REST API

Any application can connect directly using an API key:

curl -X POST http://localhost:8008/store \
  -H "X-Api-Key: sk-your-key" \
  -H "Content-Type: application/json" \
  -d '{ "text": "User prefers dark mode", "category": "preference" }'

OpenClaw

Add to your OpenClaw config.yaml:

{
  "mcpServers": {
    "neural-nexus": {
      "command": "node",
      "args": ["C:/PATH/TO/NeuralNexus/dist/src/mcp.js"]
    }
  }
}

Connection Issues

Qdrant "Connection Refused"

Qdrant is not running or the URL is wrong. Run npm run nexus to auto-start it, or check QDRANT_URL in your .env. Default port is 5304.

Retrieval Quality

Memories not showing up in Recall

RECALL_THRESHOLD may be too high. Lower it to 0.05 in .env. Also verify the correct userid is being used.

Deduplication merging too aggressively

SIMILARITY_THRESHOLD is too low. Increase it to 0.95 or 0.98 for stricter merging. Adjust in the Config page or .env.

Authentication

Recovering your password

Open .env and find NEXUS_PASSWORD. That is your dashboard login password.

Tokens stopped working after password change

Changing AUTH_SECRET invalidates all existing JWTs. Log in again via the dashboard to get a fresh token. Update any connected agents with the new token.

Diagnostics

curl http://localhost:8008/health

Check GET /audit to see why memories were merged or replaced.
